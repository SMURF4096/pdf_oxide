**Status:** Blocking ship of v0.3.35. Branch `release/v0.3.35` (forked from `origin/main` at `ec73fbc`).

## Locked decisions

Set before Phase 1 starts. Do not revisit without updating this document.

1. **Licence: MIT / Apache-2.0 only.** No MPL anywhere in the dependency closure. This rules out the Mozilla CSS stack (`cssparser`, `selectors`, `html5ever`, `lightningcss`, `stylo`). We hand-roll the CSS engine. Cost: +1 week absolute minimum, more likely +2 when the scope is this wide.
2. **Fonts: system discovery, no bundling.** When CSS says `font-family: "Helvetica Neue", Arial, sans-serif`, we walk the host's font directories (`/usr/share/fonts`, `~/.fonts`, `/System/Library/Fonts`, `C:\Windows\Fonts`, etc.) via `fontdb` and pick the best match by family + weight + style. If nothing matches, we fall back in order of CSS family list, then to the host's registered generic family for `sans-serif` / `serif` / `monospace`. No bundled fonts ship in the crate. Documented limitation: PDFs generated on a host missing the requested font look different from ones generated on a host with it — callers who need byte-stability across hosts register their own font files explicitly.
3. **CSS scope: aggressive — "Blink minus animations, filters, masks, 3D transforms".** Full CSS 2.1 visual model, Flex, Grid, tables, `@media`, `@page`, `@font-face` (local + URL), multi-column, `calc()`, `var()` / custom properties, pseudo-elements (`::before`, `::after`, `::first-line`, `::first-letter`), gradients (linear + radial), 2D transforms, `box-shadow`, `border-radius`, `text-shadow`, CSS Selectors Level 3 + parts of Level 4 (`:is`, `:where`, `:not`, `:has` — the last only within feasibility). Explicit non-goals section below lists what's excluded.

## TL;DR

Today `Pdf::from_html()` (src/api/pdf_builder.rs:337) accepts ~10 HTML tags, ignores CSS, and renders via the Markdown path. v0.3.35 replaces it with a real pipeline:

```
HTML  ─► our parser ─► DOM
CSS   ─► our tokenizer ─► AST ─► selector matcher ─► cascade + inheritance + calc/var resolution ─► computed styles
                                                      │
                                                      ▼
                                             box tree (block/inline/flex/grid/table, anonymous boxes, pseudo-elements)
                                                      │
                                                      ▼
                                             Taffy (MIT) — layout for the block/flex/grid/table modes;
                                             we own inline formatting (line boxes, floats, vertical-align,
                                             text-decoration) because Taffy doesn't do it
                                                      │
                                                      ▼
                                             paginator — fragment box tree at @page height,
                                             respecting page-break-*, orphans/widows, header/footer boxes,
                                             table <thead> repetition, multi-column balancing
                                                      │
                                                      ▼
                                             display list — fill/stroke rects, paths (for border-radius,
                                             clip-path basic), gradients, text glyph runs, images,
                                             transforms, shadows (approximated for PDF)
                                                      │
                                                      ▼
                                             pdf_oxide::writer::ContentStreamBuilder +
                                             new font subsystem (CIDFontType2 + Type0 + Identity-H +
                                             ToUnicode + subsetter) +
                                             new shading subsystem (PDF /ShadingType 2 & 3 for gradients) +
                                             new soft-mask path (for shadows & opacity)
                                                      │
                                                      ▼
                                                      PDF
```

This touches more of the write side of pdf_oxide than any release since v0.1. The font subsystem alone is worth a release. Treat the 9-week budget as tight; Phase-level risk flags below call out what will slip first if something goes wrong.

## Dependency stack (MIT / Apache-2.0 only — audited)

Direct deps added. All verified MIT, Apache-2.0, or dual. **No MPL, no GPL, no AGPL, no SSPL.** Re-run `cargo deny check licenses` after Phase 1 to prove it.

| Crate | Role | Licence | Notes |
|---|---|---|---|
| `taffy` | Block / flex / grid layout | MIT | We feed it `Style` structs we build from computed CSS. Inline layout is ours. |
| `fontdb` | System font discovery | MIT/Apache | RazrFalcon stack. Loads system fonts, matches by family/weight/style. |
| `ttf-parser` | Read TTF/OTF tables | MIT/Apache | Metrics, cmap, glyph advances. |
| `rustybuzz` | Shaping | MIT | Pure-Rust HarfBuzz port. Complex scripts, ligatures, kerning. |
| `subsetter` | Font subsetting | MIT/Apache | Used by Typst in production. We wrap, don't vendor. |
| `unicode-bidi` | BiDi for RTL / mixed-direction text | MIT/Apache | Needed for real-world Arabic/Hebrew. |
| `unicode-linebreak` | UAX #14 line-break opportunities | Apache-2.0 | Feeds the line breaker. |
| `unicode-script` | Script detection for font fallback | MIT/Apache | Pair with fontdb for per-glyph fallback. |
| `resvg` *(optional, feature-gated)* | SVG rasterisation if we decide `<img src="*.svg">` should work | MPL — ❌ |
| `usvg` + `tiny-skia` | SVG parsing only (usvg returns a tree we walk ourselves) | MPL — ❌ |

SVG-in-HTML is **out** for v0.3.35 because every viable SVG crate is MPL. We emit a placeholder for `<svg>` and `<img src="*.svg">` and document the limitation. Users who need SVG use `<img src="rasterised.png">`.

**Crates we hand-roll instead of depend on:**

- HTML parser (minimal HTML5 tokenizer — WHATWG algorithm subset sufficient for well-formed documents)
- DOM tree (read-only, flat arena-allocated, no mutation API needed)
- CSS tokenizer (CSS Syntax Module Level 3)
- CSS parser (stylesheet → rule list → declarations)
- Selector parser + matcher (CSS Selectors Level 3 + subset of L4)
- Cascade + inheritance + computed-value resolution
- `calc()` evaluator + length-unit resolution
- `var()` custom property resolution (with cycle detection)
- Inline formatting context (line boxes, vertical-align, text-decoration, floats)
- Paginator (box-tree fragmentation)
- Display-list walker
- PDF shading-dict emitter (for gradients)

Approximate new LoC: 12–18k. Most in CSS engine + inline formatting + paginator.

## Supported CSS surface (v0.3.35 commitment)

This is the contract. Everything listed renders correctly. Everything not listed either parses and is ignored (forward-compatible) or returns a parse error (surfaced in the return `Result`).

### Selectors

CSS Selectors Level 3, plus L4 where feasible:

- Type, universal (`*`), class (`.x`), ID (`#x`), attribute (`[a]`, `[a="v"]`, `[a~="v"]`, `[a|="v"]`, `[a^="v"]`, `[a$="v"]`, `[a*="v"]`, case flags)
- Combinators: descendant, child (`>`), next-sibling (`+`), subsequent-sibling (`~`)
- Structural pseudo-classes: `:root`, `:first-child`, `:last-child`, `:only-child`, `:first-of-type`, `:last-of-type`, `:only-of-type`, `:nth-child(An+B)`, `:nth-last-child`, `:nth-of-type`, `:nth-last-of-type`, `:empty`
- Logical pseudo-classes: `:is(...)`, `:where(...)`, `:not(...)`
- `:has(...)` — **best-effort**; slow path, documented caveat; v0.3.35 ships a correct implementation without the aggressive optimisations Blink has
- Pseudo-elements: `::before`, `::after`, `::first-line`, `::first-letter`
- Specificity + source order + `!important` per spec
- Forgiving parsing per L4

Out: UA interaction pseudo-classes (`:hover`, `:focus`, `:active`, `:visited`, `:checked`, form states) — no UA state in a PDF pipeline; they parse and never match.

### Box model & layout

- `display`: `block`, `inline`, `inline-block`, `flex`, `inline-flex`, `grid`, `inline-grid`, `table`, `table-row`, `table-cell`, `table-header-group`, `table-footer-group`, `table-row-group`, `table-column`, `table-column-group`, `table-caption`, `list-item`, `none`, `contents`
- `position`: `static`, `relative`, `absolute`, `fixed` (fixed behaves page-anchored in paged output — documented), `sticky` (rendered as `static` inside pagination — documented)
- `width`, `height`, `min-*`, `max-*` in all usable units
- Margins incl. `auto` for horizontal centring; collapsing margins between adjacent block siblings and parent/child (CSS 2.1 §8.3.1 full algorithm)
- Padding, border (all styles: solid, dashed, dotted, double, groove, ridge, inset, outset, none, hidden), border-radius on all four corners independently
- `box-sizing: content-box | border-box`
- `overflow-x`, `overflow-y` (`visible | hidden | clip`; `scroll`/`auto` degrade to `hidden` in paged output — documented)
- `float: left | right | none`, `clear: left | right | both | none` (v1 correctness on simple cases; `shape-outside` out)
- `z-index` with stacking contexts

### Flex & grid

- All `flex-*`, `justify-*`, `align-*`, `gap`, `row-gap`, `column-gap` properties (Taffy-backed)
- Full CSS Grid: `grid-template-columns`/`-rows` (inc. `repeat()`, `minmax()`, `fr`, `auto-fit`/`auto-fill`), `grid-template-areas`, `grid-column`, `grid-row`, `grid-gap`, `grid-auto-*`, `subgrid` — **best-effort, Taffy's subgrid is recent**
- `place-items`, `place-content`, `place-self` shorthands

### Tables

- CSS 2.1 Chapter 17 table layout: automatic and fixed
- `border-collapse: collapse | separate`, `border-spacing`
- `<thead>` repeats on every page; `<tfoot>` renders on the last table page (v0.3.35); row-level page breaks only (no mid-row splits)
- Captions positioned above/below per `caption-side`

### Typography

- `font-family` with fallback list + generic families (`serif`, `sans-serif`, `monospace`, `cursive`, `fantasy`, `system-ui`)
- `font-size` (all length units incl. `rem`, `em`, `ex`, `ch`, `vw`, `vh`, `%`); `font-size-adjust` parsed but applied as a documented approximation
- `font-weight` (100–900, `normal`, `bold`, `lighter`, `bolder`; CSS Fonts 4 inherited-weight resolution)
- `font-style` (`normal`, `italic`, `oblique` with angle)
- `font-variant-*` (small-caps, numeric variants if the font has the OpenType features — via rustybuzz feature tags)
- `line-height` (number, length, `normal`)
- `letter-spacing`, `word-spacing`
- `text-align` (`left`, `right`, `center`, `justify`, `start`, `end`); justify with spec-compliant space distribution
- `text-decoration` (`underline`, `overline`, `line-through`, combinations), `text-decoration-thickness`, `text-decoration-style` (basic solid/dotted/dashed), `text-decoration-color`
- `text-transform` (`uppercase`, `lowercase`, `capitalize`, `full-width`)
- `text-indent`
- `white-space` (`normal`, `nowrap`, `pre`, `pre-wrap`, `pre-line`, `break-spaces`)
- `word-break`, `overflow-wrap`, `hyphens: manual` (soft-hyphen honoured); `hyphens: auto` is out for v0.3.35
- `direction`, `unicode-bidi` — BiDi via `unicode-bidi` crate
- `vertical-align` (full set: `baseline`, `top`, `middle`, `bottom`, `text-top`, `text-bottom`, `sub`, `super`, `<length>`, `<percentage>`)

### Colour & backgrounds

- Colour keywords (all CSS4), `#rgb`, `#rgba`, `#rrggbb`, `#rrggbbaa`, `rgb()`, `rgba()`, `hsl()`, `hsla()`, `hwb()`, `currentColor`
- `color-mix()` — basic form
- `background-color`
- `background-image`: `url()`, `linear-gradient()`, `radial-gradient()`, `repeating-linear-gradient()`, `repeating-radial-gradient()`
- `background-size`, `background-position`, `background-repeat`, `background-clip`, `background-origin`, `background-attachment: scroll | fixed` (fixed behaves page-anchored — documented)
- Multiple backgrounds via comma-separated lists
- `opacity` (applied via PDF ExtGState `/ca`)

### Effects

- `box-shadow`: offset + blur + spread + colour + `inset`. Implementation: PDF soft masks for blur; exact for offset + spread. Multi-shadow via comma lists.
- `text-shadow`: same technique.
- `border-radius`: exact Bézier curves in content streams.
- `transform: translate | scale | rotate | skew | matrix`: 2D only. Emitted as PDF `cm` (current transformation matrix).
- `transform-origin`.
- `clip-path: inset()` and `clip-path: circle()`: exact via PDF `W*` clip. `clip-path: polygon()` via same. `clip-path: url(...)` out (requires SVG).
- `filter` — **out.** Parses, ignored, warning emitted. Exception: `filter: opacity(N)` which is equivalent to `opacity` — we alias it.

### Paged media

- `@page { size: A4 | letter | legal | <length> <length> [portrait | landscape]; margin: ... }`
- `@page :first`, `@page :left`, `@page :right`, `@page :blank`
- `@page` margin boxes (`@top-center`, `@bottom-right`, etc.) — **best-effort**; shippable subset with `counter(page)`, `counter(pages)`, and literal strings; `string-set` / `string()` out
- `page-break-before | -after | -inside`: `auto | always | avoid | left | right`
- Modern `break-*` equivalents
- `@media print` evaluates true; `@media screen` evaluates false; `@media (min-width: ...)` evaluates against the page content-box width

### Multi-column

- `column-count`, `column-width`, `columns` shorthand
- `column-gap`, `column-rule-*`
- `column-span: all | none`
- Columns balance within a page; column breaks respect `break-before | -after | -inside`

### Generated content

- `::before { content: "..." }`, `::after { content: "..." }` with:
  - Literal strings
  - `attr(name)`
  - `counter(name)`, `counters(name, sep)` — full counter model (`counter-reset`, `counter-increment`, `counter-set`)
  - `url(...)` for images
- `content: none | normal`

### `calc()`, `var()`, and at-rules

- `calc()`: +, −, ×, ÷; nested; mixed units (lengths, percentages, angles); parentheses. Division-by-zero → parse error.
- `var(--name, fallback)`: cycle detection via DFS colour marking; shadowing rules per spec.
- `@font-face`: `src: local(...), url(...)`; local resolves through fontdb; URL fetches via the same HTTP machinery we already use for OCR model downloads (feature-gated `net` dep), caches under `~/.cache/pdf_oxide/fonts/`.
- `@import url(...)` — supported for local paths and for URLs when the `net` feature is on.
- `@keyframes` — parses, ignored (we're paged; animations don't apply).

### Images

- `<img>`: JPEG, PNG, WebP (new — feature-gated on `image` crate). GIF first-frame via `image` crate.
- `srcset` / `sizes`: parse, pick the highest DPR entry that fits the rendered box.
- `data:` URIs for all the above.
- Local paths resolved relative to the HTML file's parent dir, overridable via `base_url`.
- URL fetches when `net` feature is on, same cache as `@font-face`.
- `object-fit`, `object-position`.

### Lists

- `list-style-type` (disc, circle, square, decimal, decimal-leading-zero, lower-roman, upper-roman, lower-alpha, upper-alpha, lower-greek, `none`; CJK numeric systems feature-gated by font)
- `list-style-image`, `list-style-position`

## Explicit non-goals for this release

Listed so scope discipline has teeth. Anything here is a v0.3.36+ item or never.

- **CSS filters** (`blur()`, `drop-shadow()`, `grayscale()`, etc.) beyond the `opacity()` alias.
- **3D transforms** (`translate3d`, `rotateX/Y/Z`, `perspective`, `transform-style: preserve-3d`). PDF has no 3D scene.
- **Animations & transitions.** Paged output — nothing to animate.
- **SVG.** Every viable Rust SVG crate is MPL. Deferred until we either write our own or accept MPL in a future release.
- **MathML.** Niche; no production Rust renderer at the level we'd need.
- **`hyphens: auto`.** Quality hyphenation is language-specific and heavy. `hyphens: manual` with soft hyphens ships.
- **`shape-outside`.** Rare, hard.
- **Subpixel positioning / hinting.** PDF viewers handle glyph positioning at display time.
- **CSS regions, exclusions, scroll-snap.**
- **Anchor positioning** (L1 draft).
- **Container queries.** Parse and match against the element's containing-block size; full generality deferred.
- **`view-transitions`, `@property`, `@counter-style`, `@layer`.** Parse-and-ignore with warnings.
- **JavaScript.** Static rendering only. JS-driven content routes through browser_oxide (separate track).

## Goal & success criteria

A `release/v0.3.35` HEAD that:

1. **Renders the five real-world fixtures** (defined below) to PDFs that visually match Chromium print-to-PDF within tolerance (SSIM ≥ 0.95; eye-check of every page in the test report).
2. **Supports the v0.3.35 CSS surface above** end to end — inline `style=`, `<style>` blocks, and external `<link>` + `@import`.
3. **Embeds subsetted TTF/OTF fonts** via CIDFontType2 + Type0 + Identity-H + ToUnicode. Round-trip via `extract_text` returns the input strings for Latin, Cyrillic, Greek, Arabic, Hebrew, CJK (on hosts with CJK fonts installed).
4. **Discovers system fonts** — invoices render on Linux / macOS / Windows without explicit font registration.
5. **Exposes the feature** in all four bindings (Rust, Python, Node.js, C#, Go).
6. **Zero regressions** on `~/projects/pdf_oxide_tests/` via `scripts/compare_versions.sh`.
7. **Licence audit clean** — `cargo deny check licenses` passes with MPL / GPL / AGPL / SSPL all in the deny list.
8. **Honest docs** — the `HTML_TO_PDF_GUIDE.md` lists the non-goals and links to WeasyPrint / Chromium-print as escape hatches.

Nothing ships until every box is checked.

## Phases

Each phase is a coherent commit cluster. Phases overlap only where explicitly noted. Phase codes (FONT-1, CSS-2, …) appear in commit messages so history searches stay clean.

### Week 1 — Phase FONT — PDF font subsystem

**Audit-revised.** `EmbeddedFont` + `FontSubsetter` (glyph-tracker) + `TrueTypeFont` parser + Identity-H/ToUnicode/widths-array string helpers already exist in `src/writer/font_manager.rs:840+` and `src/fonts/`. 70 unit tests pass (`tests/test_unicode_font_embedding.rs` 15 + `tests/test_cjk_script_support.rs` 55). Missing: real binary subsetter, PDF object emission, PdfWriter wiring, fontdb, rustybuzz, end-to-end round-trip. See `v0.3.35-pre-flight-audit.md` §1.

- ~~FONT-1 (ttf-parser + EmbeddedFont type)~~ — **already done** (PRE-AUDIT). Verified by running the existing tests on `release/v0.3.35` HEAD.
- **FONT-2.** Real binary subsetter via the `subsetter` crate (MIT/Apache, used by Typst). Replace `FontSubsetter`'s "track only" with `subset_font_bytes(face_bytes: &[u8], used_glyphs: &BTreeSet<u16>) -> Vec<u8>`. Wire into `EmbeddedFont`. Acceptance: 100 KB font subsets to <30 KB for typical English content.
- **FONT-3.** **Unlocking step.** New module `src/writer/font_pdf_objects.rs` with `EmbeddedFont::write_pdf_objects(&mut PdfWriter) -> FontResources` that emits five indirect objects per font: Type 0 dict (ISO 32000-1 §9.6.4), CIDFontType2 dict (§9.7.4), FontDescriptor (§9.8), FontFile2 stream (subsetted bytes), ToUnicode stream (§9.10.2). PdfWriter resource dict picks them up. Add `EmbeddedFontManager` to `PdfWriter` so callsites can switch from Base-14 to embedded fonts. Once this lands every other phase can target real fonts.
- **FONT-4.** `fontdb` system discovery. Move out of the existing `rendering` feature into a new `system-fonts` feature (default on for Python/Node.js/C#/Go bindings, default off for WASM and for Rust embedders). Walk OS font dirs lazily; cache the db on `PdfBuilder`. Match `font-family` lists with weight + style + script-coverage filtering. Deterministic tie-break: lexicographic face path. Expose `register_font_file(path)` / `register_font_bytes(name, bytes)` for explicit registration.
- **FONT-5.** `rustybuzz` shaping. Same feature treatment as FONT-4. Build `shape(text, &EmbeddedFont, size, script, direction) -> ShapedRun { glyphs: [{glyph_id, x_advance, x_offset, cluster}] }`. Complex scripts (Arabic, Devanagari) work via HarfBuzz tables already in the font.
- **FONT-6.** End-to-end round-trip — gates Phase FONT close. Render Latin / Cyrillic / Greek / Arabic / Hebrew / CJK strings using a registered font, call `Pdf::open + extract_text`, assert byte equality. CJK skipped on hosts without CJK fonts (loud skip). Add `examples/create_pdf_with_custom_font.rs`.
- **Risk:** PDF object dictionary correctness for CIDFontType2 + Type 0 + ToUnicode. Mitigation: round-trip is acceptance criterion — if extract_text returns the input string, the dict graph is correct.

**Outputs:** `src/writer/font_pdf_objects.rs` (new), `src/writer/font_discovery.rs` (new), extended `font_manager.rs`, updated `pdf_writer.rs` to register embedded fonts. `examples/create_pdf_with_custom_font.rs`.

**Compressed budget: ~3 days** (was 5 in plan v1).

### Weeks 2–4 — Phase CSS — hand-rolled CSS engine

Biggest LoC. Parallelisable at sub-phase granularity.

- **CSS-1 (week 2 d 1–2).** CSS Syntax Module Level 3 tokenizer. Zero-copy where possible. Error recovery: never panic; skip invalid; balance brackets.
- **CSS-2 (week 2 d 3).** Parser: stylesheet → rule list → qualified rules + at-rules; declaration parsing; component-value preservation for functions.
- **CSS-3 (week 2 d 4–5).** Selector parser per CSS Selectors L3 + L4 subset. Specificity calculation.
- **CSS-4 (week 3 d 1–2).** Selector matcher. Bloom filter for ancestor matching. `:has` is the expensive path — documented caveat.
- **CSS-5 (week 3 d 3–5).** Cascade + inheritance + computed-value resolution. Origin ordering, specificity, source order, `!important`. Per-element `ComputedStyle`.
- **CSS-6 (week 4 d 1).** `calc()` evaluator — mixed units, nesting, `min()`/`max()`/`clamp()`.
- **CSS-7 (week 4 d 2).** `var()` resolution with cycle detection.
- **CSS-8 (week 4 d 3).** Property parsing — one file per family. Target: every property listed in the supported surface, hand-written parsers.
- **CSS-9 (week 4 d 4).** At-rule handling: `@media`, `@page` (and margin boxes), `@font-face`, `@import`, `@keyframes` (parse-and-ignore), `@supports` (evaluate our feature set).
- **CSS-10 (week 4 d 5).** Counters: `counter-reset`, `counter-increment`, `counter-set`, `counter()`, `counters()`. Pseudo-element generation: `::before`, `::after`, `::first-line`, `::first-letter` (the last two are part of inline formatting in Phase LAYOUT; here we just generate the boxes).
- **Risk:** Property count is large. Timebox each family to 2 h; if a family is slipping, cut it to "parses, value returned, not applied" and note the scope reduction in the week-5 status update.

**Outputs:** `src/html_css/css/` with modules `tokenizer.rs`, `parser.rs`, `selectors.rs`, `cascade.rs`, `calc.rs`, `var.rs`, `properties/{length, color, font, box, flex, grid, paged, counters, transform, gradient, shadow, …}.rs`, `at_rules.rs`.

**License gate.** Run `cargo deny check licenses` at end of each week. Any MPL / GPL / AGPL / SSPL in the closure fails the week.

### Week 4 d 5 + week 5 d 1–2 — Phase HTML — parser + DOM

**Audit note.** `src/pipeline/input_parsers/html.rs` (522 lines) is a regex-based tag stripper covering ~10 tags. Keep it as a comparison oracle for the new tokenizer (assert that for the simple subset, both produce the same canonical text); do not extend it.

Can be written in parallel with late CSS phases — small enough to fit alongside CSS-9/10.

- **HTML-1.** HTML5 tokenizer — WHATWG algorithm subset (well-formed docs + the common malformed patterns). Not a full conformance implementation.
- **HTML-2.** Tree construction → flat arena DOM. Element nodes with tag, attribute map, child id list, parent id.
- **HTML-3.** Inline `style=` extraction; `<style>` block collection; `<link rel="stylesheet">` resolution (local + remote via `net` feature); `@import` inside stylesheets.
- **HTML-4.** `<img>`, `<a>` (href → PDF link annotation), basic `<picture>` srcset selection.
- **Risk:** HTML parsing edge cases are infinite. We ship the common case; malformed input gets a best-effort parse with a diagnostic in `Result::Ok` accompanied by a warnings vec.

**Outputs:** `src/html_css/html/{tokenizer, tree, dom}.rs`.

### Weeks 5–7 — Phase LAYOUT — box tree + inline + Taffy

The second-biggest phase. Inline formatting is what eats time.

- **LAYOUT-1 (week 5 d 3–4).** Box tree builder: walk DOM × computed styles → box tree with anonymous blocks / inline wrappers / table anonymous boxes per CSS 2.1 §17.2.1. Pseudo-element boxes (`::before`/`::after`) insertion.
- **LAYOUT-2 (week 5 d 5).** Map `ComputedStyle` → `taffy::Style` for block/flex/grid/table boxes. Wire Taffy's measure callback to our text measurer.
- **LAYOUT-3 (week 6 d 1–3).** Inline formatting context (the hard part, written ourselves):
  - Line boxes built from inline-level atomic items (glyph runs, replaced elements, inline-block boxes)
  - `unicode-bidi` for BiDi resolution
  - `unicode-linebreak` for UAX #14 break opportunities
  - Justify with space distribution per text-align-last
  - Vertical-align positioning; baseline / cap / x-height alignment
  - Text-decoration line computation (underline/overline/line-through with `text-decoration-skip-ink`-light heuristics)
  - `::first-line` and `::first-letter` application
  - `text-indent`, `letter-spacing`, `word-spacing`
- **LAYOUT-4 (week 6 d 4).** Floats. `float: left | right` with line-box shortening on intersecting lines. `clear`. Simple cases only; `shape-outside` is out.
- **LAYOUT-5 (week 6 d 5).** Margin collapsing per CSS 2.1 §8.3.1.
- **LAYOUT-6 (week 7 d 1–2).** Multi-column: column-count / column-width resolution, per-column fragment construction, column balancing on the last page of each multi-column flow.
- **LAYOUT-7 (week 7 d 3).** Table layout: automatic widths, fixed layout, border-collapse. `<thead>` / `<tbody>` / `<tfoot>` split hints for the paginator.
- **Risk:** Inline formatting is historically where CSS renderers spend years. Mitigations: (a) ship a correct-but-slow implementation; optimise post-release; (b) test against Blink output per line, not per paragraph; (c) timebox LAYOUT-3 to 4 days and cut `text-decoration-skip-ink` / `::first-letter` correctness first.

## Cross-cutting concerns

**Licence audit.** `deny.toml` (new or extended) lists `MPL-2.0`, `GPL-2.0`, `GPL-3.0`, `AGPL-3.0`, `SSPL-1.0` in the deny set. CI runs `cargo deny check licenses` on every PR that touches `Cargo.toml` or `Cargo.lock`.

**WASM build.** Every dep above is WASM-compatible *except* `fontdb` host-directory scanning (no filesystem in pure WASM). Solution: behind a `system-fonts` feature gate, default on for Rust/Python/Node.js/C#/Go, default off for WASM. WASM users register fonts explicitly.

**`net` feature.** Gates network-backed paths (`<link>`, `@import`, `@font-face src: url`, `<img src="https://...">`). Default on for bindings (Python user expects it to Just Work), default off for the Rust crate (library embedders opt in). Downloads cached under `~/.cache/pdf_oxide/{fonts,images,stylesheets}/`; 50 MB cache cap with LRU eviction.

**Performance.** No hard SLA. Expect 50–150 ms per page of moderate content on a 2020-era laptop. Document. Profile in week 9 if anything's a surprise. No GC pressure concerns since this is `Vec`-heavy Rust.

**Memory.** Box tree for a 500-page report can be large. v0.3.35 ships the buffered version; v0.3.36 follow-up if anyone reports OOM on enormous documents.

**Error model.** CSS parse errors non-fatal (per spec — skip invalid declarations); HTML parse errors non-fatal; font resolution fallback non-fatal with a warning; network fetch errors non-fatal with a warning (render the element without the resource). Fatal: malformed fixture causing internal invariant breakage (panic → bug, not user-facing error).

**Backwards compatibility.** `Pdf::from_html()` delegates; no breaking change. `Pdf::from_markdown()` untouched.

**Threading.** The CSS engine is single-threaded. No `Send`/`Sync` constraint on the builder. Layout of multiple documents in parallel works via separate builders.
